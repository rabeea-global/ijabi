# إيجابي — ملفات المشروع

منصّة تدعو الناس يعبّرون عن إيجابيتهم بفيديو قصير: «آني إيجابي».

## شنو بالمجلّد

| الملف | شنو يسوي | وين يُرفع |
|---|---|---|
| `index.html` | صفحة الهبوط (اللوغو + التعليمات + زرّ تليغرام) | GitHub Pages |
| `bot.py` | بوت تليغرام — يستقبل الفيديوهات ويرد آلياً | Railway / Render |
| `requirements.txt` | مكتبة البوت | مع البوت |
| `Procfile` | أمر تشغيل البوت | مع البوت |

---

## أولاً: نشر الصفحة على GitHub Pages

1. سوِّ **Repository** جديد (Public).
2. ارفع `index.html`.
   > ⚠️ **مهم:** لازم اسم الملف يكون `index.html` بالضبط — حروف صغيرة، بدون مسافات، وبدون `(1)`.
   > إذا المتصفّح نزّله باسم `index (1).html`، غيّر الاسم يدوياً قبل الرفع.
3. روح **Settings → Pages → Branch: `main` → Save**.
4. بعد دقائق، صفحتك تصير حيّة على الرابط اللي يطلع لك.
5. خذ الرابط → حوّله إلى **QR** → اطبعه على البطاقات.

## ثانياً: ربط الصفحة بالبوت

داخل `index.html`، بآخر الملف، بدّل:
```js
var TELEGRAM_BOT = "https://t.me/YOUR_BOT";
```
حُطّ مكان `YOUR_BOT` اسم بوتك (اللي أخذته من BotFather).

## ثالثاً: تشغيل البوت على Railway

1. من تليغرام: `@BotFather` → `/newbot` → خذ **التوكن**.
2. سوِّ قناة خاصة للمراجعة، أضف البوت مشرفاً، خذ **id** القناة (عن طريق `@userinfobot`).
3. داخل `bot.py` بدّل سطرين فقط:
   ```python
   TOKEN = "ضع_توكن_البوت_هنا"
   REVIEW_CHAT_ID = "ضع_id_قناة_المراجعة"
   ```
4. بـ Railway: New Project → Deploy from GitHub repo → اختر هذا المستودع.
   - Railway يقرأ `requirements.txt` و `Procfile` تلقائياً ويشغّل البوت.

---

## اختبار الحلقة كاملة
امسح البطاقة → وافق → افتح البوت → أرسل فيديو → لازم يجيك رد «تم الاستلام ✓» ويظهر الفيديو بقناة المراجعة مالتك.

ابدأ بـ **٢٠–٣٠ بطاقة** فقط، مو ١٠٠ ألف — حتى تختبر نسبة الإرسال والمراجعة قبل الطباعة الكبيرة.
